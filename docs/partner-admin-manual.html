異業合作夥伴 — 後台操作手冊

Yumie 旅館訂房後台 · 「車子保養，人也休息」異業合作專案

適用對象：旅館管理員

本手冊說明如何在 Yumie 後台操作異業合作夥伴功能。合作夥伴（如汽車保養廠、輪胎行）會拿到一組專屬推薦碼與 QR Code，把等待中的車主導引到旅館預約短時休息；車主憑碼可加贈休息時間，旅館則對每一筆實際入住（抵達）的預約回饋夥伴固定金額。本手冊涵蓋後台的合作夥伴管理、成效、結算，以及讓夥伴自己登入對帳的帳號設定。

操作前提：登入後請先在畫面右上角切換到要操作的飯店。所有合作夥伴資料、報表、結算都是依當前飯店顯示與儲存。

1. 名詞速覽

名詞 說明
推薦碼 每間合作夥伴的專屬代碼（例：A001）。客人經此碼預約即記為該夥伴帶來的來源。建立後不可修改。
休息加贈 客人憑此夥伴推薦碼預約時，額外加贈的休息分鐘數（預設 30 分）。
加時門檻 達到此預約時長（分鐘）才給加贈；預設 0＝任何時長皆贈。
每筆回饋 每一筆「抵達」的有效預約，回饋給夥伴的金額（預設 50 元）。
抵達 客人實際到店入住的訂單狀態。只有抵達的預約才計入回饋。
轉換數 經由該推薦碼成立的預約筆數（含未抵達）。
待結算 / 已結算 / 已撥款 回饋帳本的三種狀態：待結算 尚未結算 → 已結算 已產生結算單 → 已撥款 已實際付款給夥伴。

2. 合作夥伴管理

位置：設定›合作夥伴。在這裡新增、編輯、停用合作夥伴，並產生 QR Code。

2.1 新增合作夥伴

  1. 點右上角「新增夥伴」。
  2. 填寫表單欄位（見下表），其中推薦碼與店家名稱為必填。
  3. 點「儲存」。新夥伴會出現在列表，狀態為 啟用。
欄位 說明 預設
推薦碼 英文、數字、-、_，1–32 碼。系統會自動轉大寫（a001 視同 A001）。不可與既有重複，且建立後不可修改。 —
店家名稱 夥伴店家全名，例：「XX 汽車保養廠」。 —
聯絡資訊 選填，電話 / 聯絡人，方便對帳撥款時聯繫。 —
休息加贈（分鐘） 客人憑碼預約時加贈的分鐘數。 30
加時門檻（分鐘） 預約時長達此值才加贈；0＝皆贈。 0
每筆回饋（元） 每筆抵達預約回饋夥伴的金額。 50
啟用 關閉後此推薦碼立即失效（客人掃描不再加時、不再歸因）。 開

2.2 編輯合作夥伴

  1. 在列表該列點「編輯」。
  2. 修改欄位後「儲存」。
注意：編輯時推薦碼欄位是鎖定、不可修改的。若真的要換碼，請新增一組新碼，並停用舊碼（這樣歷史歸因與結算才不會混亂）。

2.3 停用合作夥伴

  1. 在列表該列點「停用」，於確認視窗按「停用」。
  2. 該夥伴狀態變為 停用。其推薦碼/QR 立即失效：客人掃描仍可訂房，但不再加贈時間、不再記為該夥伴來源。
提醒：停用不會刪除歷史。先前已成立的訂單與已產生的回饋、結算單都會保留。要重新啟用，點「編輯」把「啟用」打開即可。

3. 產生預約 QR Code

位置：設定›合作夥伴，列表每列的「QR」按鈕。這組 QR / 連結就是給夥伴張貼、讓他的客人掃描預約用的。

  1. 在該夥伴列點「QR」，跳出 QR 視窗。
  2. 視窗會顯示 QR 圖、訂房連結。連結格式為 訂房站/飯店代碼?ref=推薦碼。
  3. 點「下載 PNG」取得圖檔（檔名 referral-推薦碼.png），交給夥伴印製立牌 / 小卡 / 海報；或點「複製連結」用 LINE 等傳給夥伴。
注意：若該推薦碼目前停用，QR 視窗會顯示警告——此時掃描雖可訂房，但不會加時、不會歸因。請先啟用再發放。
掃描後會發生什麼：客人掃 QR → 進入旅館訂房頁，頁面上方出現「憑 OOO 推薦碼，本次休息加贈 N 分鐘」橫幅 → 完成預約後，該筆訂單自動記下來源與回饋金額（快照）。

4. 合作成效報表

位置：數據›合作成效。這是即時分析畫面，看各夥伴帶來的成效。

  1. 選上方日期區間（依入住日；最長 366 天）。
  2. 點「查詢」，下方表格依各推薦碼列出指標，最後一列為合計。
指標 意義
轉換數 該碼帶來、成立的預約筆數（含未抵達）。
抵達數 實際入住的預約筆數＝有效預約。
抵達成交金額 抵達訂單的成交金額合計。
平均客單價 抵達成交金額 ÷ 抵達數。
回饋金成本 抵達筆數所對應的回饋金合計（＝預估要付給夥伴的錢）。
成效 vs 結算：本頁的「回饋金成本」是即時估算，方便隨時觀察。真正要付款的權威金額，請以下一節「合作結算」產生的結算單為準。

5. 合作結算（月結）

位置：數據›合作結算。這是回饋金的權威帳本，用來每月結算並追蹤撥款。

5.1 結算流程總覽

選月份 → 核算預覽 → 一鍵結算 → 標記撥款

5.2 核算預覽

  1. 上方選月份（預設當月）。
  2. 點「核算預覽」。系統會掃描該月訂單、整理出每個夥伴待結算的筆數與金額，並顯示「待結算合計」。
  3. 可點某列的「明細」查看該夥伴的逐筆帳本分錄。
說明：核算只是整理與預覽，不會付款，可重複按。只有「抵達」的訂單會列入待結算；若某筆原本待結算的訂單後來被改成非抵達，核算會自動把它剔除。

5.3 一鍵結算

  1. 核算出有待結算金額後，點「一鍵結算」，於確認視窗按「結算」。
  2. 系統為該月有待結算的每個夥伴各產生一張結算單，狀態為 已結算，並出現在下方「結算單歷史」。
補充結算：若同一個月在你結算之後，又出現新的抵達（例如客人較晚到店補標抵達），下次再結算會為這些新筆數另外產生一張結算單（不會動到、也不會重複先前那張）。同一夥伴同月看到多張結算單是正常的。
已結算即凍結：結算單一旦產生，其涵蓋的分錄就凍結。若之後該訂單又被改為非抵達，系統不會自動沖回，需要人工處理。

5.4 標記已撥款

  1. 在「結算單歷史」找到狀態為 已結算 的單，點「標記撥款」。
  2. 視窗可填撥款備註 / 轉帳編號（選填，最多 200 字），點「確認撥款」。
  3. 該單狀態變為 已撥款，並記錄撥款時間與操作者。
防呆：已撥款的結算單不能再次標記撥款（系統會擋下），避免重複付款。

5.5 帳本明細

在「核算預覽」或「結算單歷史」任一列點「明細」，可看到該夥伴該月的逐筆回饋分錄，每筆顯示訂單 ID、入住日、金額、狀態：

狀態 意義
pending 待結算（抵達、尚未納入任何結算單）。
settled 已結算（已納入某張結算單）。
void 已作廢（該訂單非抵達，例如取消 / No-Show；不計回饋）。

6. 夥伴登入帳號（讓夥伴自己對帳）

你可以幫合作夥伴開一個登入帳號，讓他自己到夥伴對帳門戶 https://partner.yumie.flybooking.io 登入，查看屬於他自己的結算單與回饋明細。門戶是唯讀的，夥伴看不到客人個資、也看不到其他夥伴或後台任何資料。

位置：設定›合作夥伴，列表每列的「帳號」按鈕；列上「登入帳號」欄顯示目前狀態。

6.1 建立登入帳號

  1. 在該夥伴列點「帳號」。
  2. 輸入夥伴的 Email（作為登入帳號，須未被使用）。
  3. 密碼可留空：系統會產生一組密碼設定連結，把連結轉交夥伴自行設定；或直接幫他設一組初始密碼。
  4. 點「建立帳號」。狀態變為 已啟用，夥伴即可登入門戶。
提醒：若視窗顯示密碼設定連結，用「複製連結」轉交夥伴（連結會導到門戶登入頁）。

6.2 帳號狀態

狀態 意義
未建立 尚未開帳號。
已啟用 夥伴可正常登入對帳。
建立中／待啟用 建立過程中斷的暫態；可再按「重設密碼」或聯絡工程處理。
已停用 夥伴無法登入（即時生效）；可「重新啟用」。

6.3 重設密碼 / 停用 / 重新啟用

  1. 重設密碼：帳號視窗點「重設密碼」，產生重設連結轉交夥伴。
  2. 停用帳號：點「停用帳號」→ 夥伴立即無法登入（保留帳號，可再啟用）。
  3. 重新啟用：已停用的帳號點「重新啟用」即可恢復登入。

6.4 夥伴在門戶看到什麼

夥伴登入後，可選月份查看自己的：

  • 頂部三個數字：待結算 / 已結算 / 已撥款 金額。
  • 結算單：每張的筆數、金額、狀態（已結算／已撥款）、撥款時間與備註。
  • 明細：逐筆入住日、回饋金額、狀態（pending／settled／void）。
對帳一致性：夥伴門戶的數字＝後台「合作結算」同一夥伴同月的權威金額；你按「標記撥款」後，夥伴頁會看到該單變 已撥款。
換 Email：目前不支援直接改帳號 Email。要換 email 請聯絡工程移除舊帳號後重新建立（停用只是停權、不會釋放 email）。

7. 重要規則與注意事項

1. 推薦碼建立後不可修改。要換碼＝新增新碼＋停用舊碼。
2. 回饋金只算「抵達」的有效預約；未抵達、取消、No-Show 都不算。
3. 客人完成預約當下，會把當時的回饋金額快照在訂單上。之後你調整夥伴的「每筆回饋」金額，不會影響已成立的歷史訂單，只影響之後的新訂單。
4. 加時採擇優取最大、不疊加：若客人同時符合「會員等級加時」與「推薦碼加時」，取較多的那個（同分鐘數時以會員為準）。
5. 結算單已結算即凍結；已撥款不可重複。同一夥伴同月可能有多張結算單（補充結算）。
6. 停用的推薦碼，客人掃描仍可訂房，但不加時、不歸因。

8. 常見問題 FAQ

客人說掃了 QR，但訂房頁沒看到加贈時間？

常見原因：① 該推薦碼目前停用；② 客人選的預約時長未達加時門檻；③ QR/連結被改過、推薦碼格式不正確。請到「設定 › 合作夥伴」確認該碼為啟用、門檻設定，並重新用「QR」按鈕產生連結給夥伴。

「合作成效」的回饋金成本，和「合作結算」的金額對不起來？

「合作成效」是即時估算、會隨訂單狀態變動；「合作結算」是結算當下凍結的權威金額。若該月還沒結算、或結算後又有新抵達（補充結算），兩者短期內有差是正常的。付款請以結算單為準。

已經結算（甚至撥款）後，客人才取消或變成 No-Show 怎麼辦？

已結算的分錄凍結、不會自動沖回。這種情況請以人工方式在下次與夥伴對帳時調整。（結算前的狀態變動則會由「核算」自動作廢，不需手動處理。）

某個夥伴這個月明明有客人，卻找不到他的結算單？

請確認：① 是否已按「一鍵結算」；② 該夥伴本月是否有抵達的訂單（只有抵達才結算）；③ 月份是否選對（結算依入住日所屬月份）。可先用「核算預覽」看該夥伴是否有待結算金額。

QR Code 裡的連結是指向哪個網站？

預設指向正式訂房站。此網址由系統設定 VITE_GLOB_BOOKING_BASE_URL 控制；若測試環境要指向測試站，請工程同仁於對應環境設定覆寫。一般營運不需更動。

看不到「合作夥伴 / 合作成效 / 合作結算」選單？

① 確認已用有權限的旅館管理員帳號登入；② 確認右上角已選到正確的飯店。若仍看不到，請聯繫系統管理者確認帳號權限與選單設定。

夥伴說登入不了對帳門戶？

請確認：① 該夥伴的「登入帳號」狀態為 已啟用（在「設定 › 合作夥伴」該列查看）；② 用的是建立時填的 Email；③ 忘記密碼可在「帳號」視窗按「重設密碼」產生連結轉交夥伴。若帳號是 已停用，請先「重新啟用」。

夥伴頁全是 0，但後台「合作結算」看得到他的金額？

正常情況兩邊應一致（同一權威金額）。若夥伴頁整頁是 0、但後台確實有該夥伴的結算單，多半是系統面問題（非「沒資料」），請聯絡工程檢查，毋須自行調整資料。
